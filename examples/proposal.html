Example Group Project Proposal

Team Name: swampy-eliot
Application Name: Blank Slate
Technology Used:
  • Front End: React
  • Back End: Java - Spring
  • Database: MySQL
Initial Features:
  • A user will have the ability to post a project idea, including screenshots / images
  • A reviewer (or commenter) will be able to provide feedback
  • A reviewer (or commenter) will be able to mark that they are interested in helping
  • Tags will be added to the project idea, for languages or specific needs
  • A reviewer (or commenter) can search by tags to find projects
  • A project can be marked as closed or no longer requesting feedback, to limit clutter
  • A user will have to create an account to create a project or post feedback / comments
Soruce Control: Github
Deployment: Heroku
How does this meet the requirements:
MySQL will hold user information, project information, and comments.
Java - Spring will assist with user registration, authentication, and allow for the data from the database to be exposed through an API.
React will be the primary user interface for users, and will allow them to navigate to sections based on authentication.
